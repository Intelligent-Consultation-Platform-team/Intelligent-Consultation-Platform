# 前端迭代待办清单

## 一、迭代目标（建议 2 周）

- 目标1：完成核心流程联调闭环（登录→挂号→就诊→住院→缴费）
- 目标2：提升系统稳定性（异常处理、权限兜底、防重复提交）
- 目标3：统一交互体验（加载态/空态/错误态、表单校验、提示文案）

---

## 二、任务清单（按模块 + 优先级 + 预估工时）

> 说明：
> - 优先级：P0（必须）、P1（重要）、P2（优化）
> - 工时按“人天”粗估，可按团队情况调整

### A. API 与联调（P0）

1. **统一 API 返回结构适配**
   - 内容：统一 `success/code/message/data` 处理逻辑
   - 工时：0.5~1 人天
   - 验收：所有页面请求走统一解析，不再页面内重复判断

2. **全局错误处理**
   - 内容：401/403/500/超时统一提示与跳转策略
   - 工时：0.5 人天
   - 验收：模拟各类错误时表现一致

3. **接口联调问题台账**
   - 内容：逐页面记录字段不一致、缺字段、分页参数差异
   - 工时：1 人天（穿插进行）
   - 验收：形成可追踪清单并逐项关闭

### B. 权限与安全（P0）

4. **按钮级权限控制**
   - 内容：除路由守卫外，控制“新增/删除/编辑/审核”等操作按钮显示
   - 工时：1 人天
   - 验收：不同角色登录时操作按钮符合预期

5. **登录态一致性**
   - 内容：token/session 过期处理、退出登录清理、刷新后恢复
   - 工时：0.5 人天
   - 验收：过期自动回登录；退出后不可访问受限页面

### C. 表单与交互质量（P0-P1）

6. **核心表单校验补齐**
   - 内容：注册、挂号、住院、充值/缴费校验规则统一
   - 工时：1~1.5 人天
   - 验收：非法输入可拦截，提示明确

7. **防重复提交**
   - 内容：提交按钮 loading + 禁用；请求未完成不可再次点击
   - 工时：0.5 人天
   - 验收：快速连点不会重复写入数据

8. **统一消息提示规范**
   - 内容：成功/失败提示文案模板化
   - 工时：0.5 人天
   - 验收：全站提示风格一致

### D. 列表页体验完善（P1）

9. **通用列表能力补齐**
   - 内容：分页、搜索、重置、排序（按页面逐步落地）
   - 工时：1.5~2 人天
   - 验收：管理类页面都可稳定筛选和翻页

10. **三态 UI 标准化**
    - 内容：加载态、空态、错误态（含重试）
    - 工时：1 人天
    - 验收：所有核心列表都有三态显示

### E. 工程化与维护性（P1-P2）

11. **环境配置拆分**
    - 内容：dev/test/prod 的 baseURL 与配置分离
    - 工时：0.5 人天
    - 验收：不同环境打包后可正确访问对应后端

12. **公共组件抽取**
    - 内容：查询栏、分页容器、确认弹窗、空状态组件
    - 工时：1~1.5 人天
    - 验收：重复代码减少，后续页面开发提速

13. **状态管理收敛（可选）**
    - 内容：把用户信息/角色/会话收口到统一状态管理
    - 工时：1 人天
    - 验收：页面不再散落读取 localStorage

### F. 测试与验收（P1）

14. **核心链路冒烟测试**
    - 内容：3角色分别验证关键业务流
    - 工时：1 人天
    - 验收：输出测试记录与问题闭环

15. **发布前回归清单**
    - 内容：权限、表单、接口、异常、兼容性检查
    - 工时：0.5 人天
    - 验收：形成发布 checklist

---

## 三、推荐排期（两周版）

### 第 1 周（保交付）
- 完成：A1-A3、B4-B5、C6-C8
- 目标：业务可稳定联调，权限和异常可控

### 第 2 周（提质量）
- 完成：D9-D10、E11-E12、F14-F15
- 可选：E13（状态管理收敛）
- 目标：体验明显提升，可进入验收/演示阶段

---

## 四、里程碑验收标准

- **M1（联调通过）**：核心接口成功率 > 95%，无阻塞级问题
- **M2（权限稳定）**：3角色越权操作均被拦截
- **M3（体验达标）**：核心页面具备加载/空/错三态，表单校验完善
- **M4（可发布）**：冒烟通过、回归清单通过、已知问题可控
